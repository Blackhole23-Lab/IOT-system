## 1. 项目概述

智慧课堂系统（IOT-System Smart Classroom）是一个现代化的教育信息化平台，为教师、学生和管理员提供统一的在线教学、考试、成绩管理和用户管理解决方案。

### 1.1 核心功能

| 功能模块 | 管理员端 | 教师端 | 学生端 | 描述 |
|---------|--------|------|------|------|
| **认证系统** | 用户CRUD | 登录/注册 | 登录/注册 | JWT令牌，支持微信模拟登录 |
| **用户管理** | 管理所有用户 | - | - | 创建、编辑、删除、统计 |
| **演示课堂** | - | 上传PDF、翻页、标注、激光笔 | 同步查看、跟随演示 | 实时WebSocket同步 |
| **在线考试** | - | 创建考试、发布、批改、统计 | 答题、自动提交、查看成绩 | 自动判题+手动批改 |
| **成绩管理** | 查看全部数据 | 查看所有学生成绩、批改主观题 | 查看个人成绩、逐题分析 | 平均分、排名统计 |
| **出勤签到** | - | 查看教室出勤记录 | 自动记录上课出勤 | 按日期按房间去重 |

### 1.2 技术栈

**前端**:
- React 18 + TypeScript
- Vite 构建工具
- React Router v6 路由管理
- TailwindCSS UI框架
- Lucide React 图标库
- Socket.io 客户端 (WebSocket)
- qrcode.react 二维码生成
- pdf.js + Reveal.js (PDF演示)

**后端**:
- Node.js + Express.js
- Socket.io 服务器
- JWT 身份认证
- Multer 文件上传
- 内存存储 (Map/数组，无数据库)

**部署**:
- PM2 进程管理
- Docker 容器化 (可选)
- 环境变量配置

## 3. 认证系统

### 3.1 用户角色与权限

系统支持三种用户角色，各具不同权限和默认跳转路径：

| 角色 | 值 | 权限 | 默认跳转 |
|------|-----|------|---------|
| 管理员 | admin | 用户CRUD，查看全部数据统计 | /admin/dashboard |
| 教师 | teacher | 创建房间，批改卷子，查看学生 | /teacher/dashboard |
| 学生 | student | 加入课堂/考试，查看个人记录 | /student/dashboard |

### 3.2 JWT令牌

服务器使用 JWT 进行身份认证，令牌包含用户信息和过期时间。

**令牌结构**:
```json
{
  "id": "u1",
  "username": "teacher",
  "role": "teacher",
  "name": "王老师",
  "iat": 1234567890,
  "exp": 1234588490
}
```

**令牌有效期**: 8小时

**存储位置**: 浏览器 `localStorage` 中的 `iot_token` 键

**验证方式**: HTTP 请求头 `Authorization: Bearer <token>`

### 3.3 认证流程

#### 账号登录

1. 用户输入用户名和密码
2. 前端发送 POST 请求到 `/api/auth/login`
3. 后端校验凭证，生成 JWT
4. 前端保存令牌并更新认证状态
5. 路由守卫检查角色，重定向到对应 Dashboard

#### 账号注册

1. 用户填写用户名、密码、角色、显示名称
2. 注册时仅支持 `teacher` 和 `student` 角色
3. 前端发送 POST 请求到 `/api/auth/register`
4. 后端创建用户，生成 JWT
5. 自动登录，重定向到 Dashboard

#### 微信模拟登录

1. 用户点击"微信扫码登录"
2. 显示二维码扫描动画（模拟）
3. 用户确认身份、输入显示名称、选择角色
4. 前端自动生成唯一的微信用户名（`wx_` 前缀）
5. 调用 `/api/auth/register` 自动注册
6. 登录成功后重定向回原始页面

**微信用户特征**:
- 用户名格式: `wx_[随机6位字符]`
- 默认密码: `[用户名]_wechat_auto`（由前端自动生成）
- 用户可自定义显示名称，默认为"微信同学"或"微信老师"

### 3.4 URL重定向参数

登录页支持 `?redirect=` 参数，登录/注册成功后自动跳转到指定路径。

**用途**:
- QR码深度链接：`/login?redirect=/test/student/ABCD1234`
- 未登录时自动保存原始路径

**示例**:
```
GET /login?redirect=%2Ftest%2Fstudent%2FABCD1234
```

登录成功 → 自动跳转 `/test/student/ABCD1234`

### 3.5 路由守卫

前端在 `App.tsx` 中定义了多个守卫组件：

**PrivateRoute**
- 检查用户是否登录（`user !== null`）
- 未登录时重定向到 `/login?redirect=[原路径]`

**TeacherOnly**
- 检查用户角色是否为 `teacher`
- 非教师重定向到 `/dashboard`

**StudentOnly**
- 检查用户角色是否为 `student`
- 非学生重定向到 `/dashboard`

**AdminOnly**
- 检查用户角色是否为 `admin`
- 非管理员重定向到 `/dashboard`

**DashboardRedirect**
- `/dashboard` 路由根据用户角色重定向：
  - admin → `/admin/dashboard`
  - teacher → `/teacher/dashboard`
  - student → `/student/dashboard`

### 3.6 管理员权限控制

#### 服务器端 - requireAdmin 中间件

所有管理员 API 前都挂载 `requireAdmin` 中间件：

```javascript
function requireAdmin(req, res, next) {
  const authHeader = req.headers.authorization;
  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return res.status(401).json({ success: false, message: '未登录' });
  }
  try {
    const payload = jwt.verify(authHeader.slice(7), getSecret());
    if (payload.role !== 'admin') {
      return res.status(403).json({ success: false, message: '需要管理员权限' });
    }
    req.admin = payload;
    next();
  } catch {
    return res.status(401).json({ success: false, message: 'Token 已过期' });
  }
}
```

#### 客户端 - AdminOnly 组件

路由在 `App.tsx` 中使用 `AdminOnly` 守卫：

```jsx
<Route path="/admin/dashboard" element={
  <AdminOnly><AdminDashboard /></AdminOnly>
} />
```

#### 受保护的操作

- **超级管理员不可删除**: 尝试 `DELETE /api/auth/admin/users/admin` 返回 403
- **管理员角色不可修改**: 根用户 `admin` 的角色锁定为 `admin`，无法改为 teacher/student

### 4.2 管理员后台 (AdminDashboard.tsx)

**路径**: `/admin/dashboard`

**权限**: 仅 `admin` 角色可访问

**功能**:
- 用户管理：创建、查看、编辑、删除用户
- 四个标签页：总览、学生、教师、管理员
- 搜索功能（姓名/用户名/邮箱）
- 统计卡片：总用户数、学生数、教师数、总答卷数
- 每个用户显示统计：考试次数、上课次数、平均分、待批卷数

**核心状态**:
```typescript
const [tab, setTab] = useState<Tab>('overview')           // 当前标签

```

**API 调用**:
- `GET /api/auth/admin/users` — 获取所有用户
- `POST /api/auth/admin/users` — 创建用户
- `PUT /api/auth/admin/users/:username` — 更新用户
- `DELETE /api/auth/admin/users/:username` — 删除用户
- `GET /api/test/submissions` — 获取答卷统计
- `GET /api/teach/attend` — 获取上课记录

**用户编辑**:
- 支持修改: 姓名、邮箱、密码（可选）、角色
- 超级管理员 `admin` 的角色无法改为其他值
- 超级管理员 `admin` 的删除按钮被禁用

**用户创建**:
- 必填: 用户名（3-20位）、密码（至少6位）、角色
- 可选: 姓名、邮箱
- 支持创建 teacher/student/admin 三种角色

## 5. 后端 API

### 5.1 认证模块 (/api/auth/*)

#### POST /api/auth/login

**描述**: 账号登录

**请求体**:
```json
{
  "username": "teacher",
  "password": "teacher123"
}
```

**响应** (成功, 200):
```json
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "u1",
    "username": "teacher",
    "role": "teacher",
    "name": "王老师"
  }
}
```

**响应** (失败, 401):
```json
{
  "success": false,
  "message": "用户名或密码错误"
}
```

#### POST /api/auth/register

**描述**: 注册账号（仅支持 teacher/student 角色，admin 角色由管理员创建）

**请求体**:
```json
{
  "username": "student1",
  "password": "password123",
  "role": "student",
  "name": "李同学",
  "email": "student1@example.com"
}
```

**验证**:
- username: 3-20 字符，必填
- password: 至少 6 位，必填
- role: 仅 'teacher' 或 'student'，必填
- name: 可选，默认为 username + 老师/同学
- email: 可选

**响应** (成功, 200):
```json
{
  "success": true,
  "message": "注册成功",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "u6",
    "username": "student1",
    "role": "student",
    "name": "李同学"
  }
}
```

**响应** (失败, 409 用户名重复):
```json
{
  "success": false,
  "message": "用户名已存在"
}
```

#### GET /api/auth/me

**描述**: 验证令牌，获取当前用户信息

**请求头**:
```
Authorization: Bearer <token>
```

**响应** (成功, 200):
```json
{
  "success": true,
  "user": {
    "id": "u1",
    "username": "teacher",
    "role": "teacher",
    "name": "王老师"
  }
}
```

**响应** (失败, 401):
```json
{
  "success": false,
  "message": "Token 已过期，请重新登录"
}
```

#### GET /api/auth/students

**描述**: 获取所有学生账号列表（教师用）

**请求头**:
```
Authorization: Bearer <teacher_token>
```

**响应** (200):

## 7. 二维码系统

### 7.1 二维码生成与显示

前端组件 `RoomQR.tsx` 使用 `qrcode.react` 库生成二维码。

**生成 URL 格式**:
```
https://your-domain.com/[system]/student/[roomCode]
```

其中：
- `system`: 'teach' 或 'test'
- `roomCode`: 房间码，如 'ABCD1234'

**示例**:
```
https://classroom.local:5173/test/student/ABCD1234
https://classroom.local:5173/teach/student/ROOM5678
```

### 7.2 二维码扫描与登录流程

#### 场景 1: 学生已登录

1. 学生扫描二维码
2. 直接跳转到 `/test/student/ABCD1234` 或 `/teach/student/ROOM5678`
3. `PrivateRoute` 守卫验证登录状态
4. 加载对应页面

#### 场景 2: 学生未登录

1. 学生扫描二维码
2. 跳转到 `/test/student/ABCD1234`（未登录）
3. `PrivateRoute` 守卫拦截，重定向到 `/login?redirect=%2Ftest%2Fstudent%2FABCD1234`
4. 学生登录或注册
5. 登录成功后自动重定向回 `/test/student/ABCD1234`
6. 加载对应页面

**流程图**:
```
扫描二维码
    ↓
跳转到 /[system]/student/[code]
    ↓
    ├─ 已登录 → 加载页面
    └─ 未登录 → 重定向到 /login?redirect=...
         ↓
     登录/注册
         ↓
     自动返回 /[system]/student/[code]
```

### 7.3 深度链接与重定向参数保持

- 二维码 URL 携带房间码参数
- 登录时 `redirect` 参数自动保存到 URL
- 登录成功后自动跳转回原始房间

## 8. 数据模型

### 8.1 用户 (User)

```typescript
interface User {
  id: string;                    // 用户ID，格式 u{number}
  username: string;              // 用户名，唯一，3-20字符
  password: string;              // 明文密码（生产环境应哈希）
  role: 'admin' | 'teacher' | 'student';
  name: string;                  // 显示名称
  email?: string;                // 邮箱
  createdAt: string;             // 创建时间，ISO 8601
}
```

**存储**: 服务器内存中的 Map，键为 username

**初始数据**:
```javascript
[
  ['teacher',  { id: 'u1', username: 'teacher',  password: 'teacher123', role: 'teacher', ... }],
  ['teacher2', { id: 'u2', username: 'teacher2', password: 'teacher123', role: 'teacher', ... }],
  ['student',  { id: 'u3', username: 'student',  password: 'student123', role: 'student', ... }],
  ['student2', { id: 'u4', username: 'student2', password: 'student123', role: 'student', ... }],
  ['admin',    { id: 'u5', username: 'admin',    password: 'admin123',   role: 'admin',   ... }],
]
```

### 8.2 提交记录 (Submission)

```typescript
interface Submission {
  id: string;                    // 答卷ID
  userId: string;                // 学生用户ID
  examId: string;                // 考试ID
  score: number;                 // 得分
  totalScore: number;            // 总分
  status: 'pending' | 'completed'; // 批改状态
  submittedAt: string;           // 提交时间
  answers: Record<string, AnswerDetail>; // 逐题答卷
}

interface AnswerDetail {
  answer: unknown;               // 学生答案
  score: number | null;          // 题目得分（主观题可能为null）
  maxScore: number;              // 满分
  questionText?: string;
  questionType?: string;
  options?: string[];            // 选项（单选/多选）
  correctAnswer?: string | string[]; // 正确答案
}
```

### 8.3 出勤记录 (Attendance)

```typescript
interface Attendance {
  id: string;                    // 记录ID

  roomCode: string;              // 房间码
  date: string;                  // 日期 (YYYY-MM-DD)
  joinedAt: string;              // 加入时间

```

**去重规则**: 同一学生、同一房间码、同一天只记录一次

```typescript
interface Room {

  createdAt: string;             // 创建时间
  students: string[];            // 加入的学生ID列表
  active: boolean;               // 是否活跃
}
```

## 9. 常见操作指南

### 9.1 教师创建与开启课堂

1. **登录**: 使用教师账号登录
2. **进入首页**: 自动重定向到 `/teacher/dashboard`
3. **生成房间码**:
   - 选择"演示课堂"或"在线考试"
   - 点击系统卡片自动生成房间码
   - 可点击"换一个"重新生成
4. **分享房间**:
   - 显示房间码（6位）给学生输入
   - 扫描二维码进行深度链接
5. **进入控制台**:
   - 点击"进入演示课堂控制台"或"进入考试系统控制台"
   - 导航到 `/teach/teacher/:code` 或 `/test/teacher/:code`

### 9.2 学生加入课堂

#### 方式 1: 输入房间码

1. 登录或注册
2. 重定向到学生首页或选择加入
3. 输入教师分享的房间码
4. 点击"加入"

#### 方式 2: 扫描二维码

1. 使用手机相机或二维码应用扫描
2. 自动跳转到房间页面
3. 若未登录，先登录/注册，然后自动进入

#### 方式 3: 直接访问 URL

1. 访问 `/test/student/ABCD1234` 或 `/teach/student/ROOM5678`
2. 若未登录，重定向到 `/login?redirect=...`
3. 登录后自动进入房间

### 9.3 教师批改学生答卷

1. **进入教师首页**
2. **查看学生档案** (两种方式):
   - 点击"最新答卷"列表中的某个答卷
   - 点击"学生名册"中的某个学生
3. **在学生档案页面**:
   - 查看考试列表
   - 点击某个待批改的考试展开详情
   - 在主观题下输入评分
   - 点击"保存评分"

### 9.4 管理员管理用户

1. **登录**: 使用管理员账号 `admin / admin123`
2. **进入管理后台**: 自动重定向到 `/admin/dashboard`
3. **切换标签页**:
   - 总览：显示学生和教师列表
   - 学生：仅显示学生
   - 教师：仅显示教师
   - 管理员：仅显示管理员
4. **创建用户**:
   - 点击"新建用户"
   - 填写用户名、密码、角色（必填）、姓名、邮箱（可选）
   - 点击"创建"
5. **编辑用户**:
   - 点击用户行的编辑按钮
   - 修改姓名、邮箱、密码、角色
   - 点击"保存"
6. **删除用户**:
   - 点击用户行的删除按钮
   - 确认删除（超级管理员 `admin` 不可删除）
7. **搜索用户**:
   - 在"学生/教师/管理员"标签页中使用搜索框
   - 支持按姓名、用户名、邮箱搜索

## 12. 常见问题与故障排查

### 12.1 用户登录问题

**问题**: 登录后无法进入 Dashboard
- 检查是否启用了正确的路由守卫
- 验证令牌是否存储到 `localStorage`
- 检查浏览器控制台的网络请求

**问题**: Token 过期
- 当前令牌有效期为 8 小时
- 过期后自动清除，需重新登录
- 生产环境考虑实现 refresh token

### 12.2 房间码生成问题

**问题**: 房间码无法生成
- 确保用户已登录且为 `teacher` 角色
- 检查后端 `/api/teach/rooms/generate` 是否正常运行
- 查看浏览器控制台错误日志

### 12.3 WebSocket 连接问题

**问题**: Socket.io 连接失败
- 确保后端 Socket.io 服务已启动
- 检查浏览器网络标签中的 WebSocket 连接状态
- 验证代理配置（Nginx/Apache）支持 WebSocket upgrade

**问题**: 实时事件未收到
- 检查事件名称拼写是否正确
- 验证客户端是否加入了正确的命名空间（/teach 或 /test）
- 查看后端日志是否有错误

### 12.4 答卷批改问题

**问题**: 批改后分数未更新

## 13. 性能优化建议

### 13.1 前端优化

- 使用 React.memo 避免不必要的重新渲染
- 延迟加载大型组件（如 PDF 查看器）
- 缓存 API 响应数据

### 13.2 后端优化

- 将内存数据存储迁移到数据库（MongoDB/PostgreSQL）
- 实现查询缓存（Redis）
- 使用分页查询大数据集

### 13.3 网络优化

- 压缩 API 响应 (gzip)
- 启用 HTTP/2 和 SSL/TLS
- 使用 CDN 加速静态资源

---

## 14. 安全建议

### 14.1 身份认证

- 不要在生产环境中使用硬编码的 JWT 密钥
- 定期轮换密钥
- 实现 CSRF 防护

### 14.2 数据安全

- 密码使用 bcrypt 或 Argon2 哈希，不存储明文
- 使用 HTTPS 加密传输
- 定期备份数据

### 14.3 API 安全

- 实现速率限制防止暴力破解
- 验证所有输入数据（XSS/SQL 注入防护）
- 使用 CORS 白名单
