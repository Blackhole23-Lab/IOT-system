# 在线考试系统 - 完整项目说明文档

## 📋 项目概述

**项目名称**：在线考试系统（Online Exam System）
**开发语言**：PHP 8.3 + SQLite 3
**前端框架**：Bootstrap 5 + 原生JavaScript
**容器化**：Docker + Docker Compose
**开发时间**：2026年3月
**项目类型**：Web应用 - 教育考试平台

## 🎯 项目特点

### 核心优势
- ✅ **零依赖部署**：无需Composer，无需框架，开箱即用
- ✅ **容器化支持**：完整Docker配置，一键启动
- ✅ **轻量级数据库**：SQLite数据库，无需额外安装
- ✅ **安全设计**：PDO预处理、密码加密、XSS防护
- ✅ **响应式界面**：Bootstrap 5，支持移动端访问

### 适用场景
- 中小型学校在线考试
- 企业内部培训考核
- 在线教育平台
- 技能测评系统
- 小规模竞赛系统（<1000人）

## 💡 功能模块详解

### 1. 用户系统
#### 用户注册
- 用户名验证（唯一性检查）
- 邮箱格式验证（唯一性检查）
- 密码强度要求（最少6位）
- 密码确认验证
- 自动分配学生角色

#### 用户登录
- 用户名/邮箱登录
- 密码验证（password_verify）
- Session会话管理
- 记住登录状态
- 登录失败提示

#### 权限控制
- **管理员（admin）**：完全权限
  - 题库管理
  - 考试管理
  - 成绩管理
  - 用户管理

- **学生（student）**：受限权限
  - 参加考试
  - 查看成绩
  - 个人信息

### 2. 题库管理（管理员）

#### 支持的题型
1. **单选题（single）**
   - 2-6个选项
   - 单一正确答案
   - 自动判分

2. **多选题（multiple）**
   - 2-6个选项
   - 多个正确答案
   - 完全正确才得分

3. **判断题（judge）**
   - True/False选择
   - 自动判分

4. **编程题（code）**
   - 大文本输入框
   - 需要人工阅卷
   - 支持代码高亮

5. **简答题（essay）**
   - 大文本输入框
   - 需要人工阅卷

#### 题库操作
- **手动添加**：表单录入，动态显示字段
- **批量导入**：CSV/JSON文件上传
- **编辑题目**：修改题目内容和选项
- **删除题目**：支持单个删除
- **分页显示**：每页20题，提升加载速度

#### CSV导入格式
```csv
题型,题目内容,选项1,选项2,选项3,选项4,选项5,选项6,答案,分值
single,PHP是什么?,编程语言,数据库,操作系统,浏览器,,,1,5
multiple,以下哪些是Web技术?,HTML,CSS,JavaScript,Python,,,1,2,3,10
judge,PHP是开源的,,,,,,,true,5
code,编写一个冒泡排序算法,,,,,,,参考答案代码,20
essay,简述MVC设计模式,,,,,,,参考答案文本,15
```

### 3. 考试管理（管理员）

#### 创建考试
- 设置考试标题
- 设置考试时长（分钟）
- 从题库选择题目（多选）
- 自动计算总分
- 保存考试配置

#### 考试编辑
- 修改考试信息
- 调整题目组合
- 更新考试时长

#### 考试删除
- 删除考试配置
- 级联删除考试-题目关联
- 保留学生提交记录

### 4. 在线考试（学生）

#### 考试列表
- 显示所有可参加的考试
- 显示考试信息（标题、时长、总分）
- 显示参加状态（未参加/已完成）
- 一键开始考试

#### 答题界面
- **倒计时功能**：实时显示剩余时间
- **自动提交**：时间到自动提交
- **防止离开**：beforeunload警告
- **题目展示**：根据题型动态渲染
- **答案保存**：实时记录答案
- **手动提交**：确认对话框

#### 答题体验
- 单选题：Radio按钮
- 多选题：Checkbox复选框
- 判断题：True/False按钮
- 编程题：大文本框（支持代码）
- 简答题：大文本框

### 5. 成绩管理

#### 学生端
- 查看已参加的考试
- 显示考试得分
- 显示总分和得分率
- 显示考试状态（已完成/待阅卷）

#### 管理员端
- 查看所有学生提交记录
- 显示学生信息、考试信息、得分
- 导出CSV格式成绩单
- 主观题人工阅卷（grade.php）

#### 自动判分规则
- **单选题**：选择正确得满分，否则0分
- **多选题**：完全正确得满分，否则0分
- **判断题**：选择正确得满分，否则0分
- **编程题**：标记为"待阅卷"，需人工评分
- **简答题**：标记为"待阅卷"，需人工评分

## 🗄️ 数据库设计

### 表结构

#### 1. users（用户表）
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    email TEXT UNIQUE NOT NULL,
    password TEXT NOT NULL,
    role TEXT DEFAULT 'student',  -- admin/student
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. questions（题库表）
```sql
CREATE TABLE questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type TEXT NOT NULL,  -- single/multiple/judge/code/essay
    question_text TEXT NOT NULL,
    options TEXT,  -- JSON格式存储选项
    answer TEXT NOT NULL,
    score INTEGER DEFAULT 5,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 3. exams（考试表）
```sql
CREATE TABLE exams (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    duration INTEGER NOT NULL,  -- 分钟
    total_score INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 4. exam_questions（考试-题目关联表）
```sql
CREATE TABLE exam_questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    exam_id INTEGER NOT NULL,
    question_id INTEGER NOT NULL,
    FOREIGN KEY (exam_id) REFERENCES exams(id),
    FOREIGN KEY (question_id) REFERENCES questions(id)
);
```

#### 5. submissions（提交记录表）
```sql
CREATE TABLE submissions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    exam_id INTEGER NOT NULL,
    answers TEXT,  -- JSON格式存储答案
    score INTEGER DEFAULT 0,
    status TEXT DEFAULT 'pending',  -- pending/completed
    submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (exam_id) REFERENCES exams(id)
);
```

## 🚀 部署方案

### 方案一：Docker部署（推荐）

#### 系统要求
- Docker 20.10+
- Docker Compose 2.0+
- 2GB+ 内存
- 10GB+ 磁盘空间

#### 快速启动
```bash

chmod +x 一键安装.sh
./一键安装.sh

# 2. 访问系统

```

#### 手动部署
```bash

docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 方案二：传统部署

#### 环境要求
- PHP 8.0+
- Apache/Nginx
- SQLite 3扩展
- PDO扩展

#### 部署步骤
```bash
# 1. 安装PHP和Apache
sudo apt update
sudo apt install php8.3 php8.3-sqlite3 php8.3-pdo apache2

# 2. 复制文件到Web目录
sudo cp -r exam_system /var/www/html/

# 3. 设置权限
sudo chown -R www-data:www-data /var/www/html/exam_system
sudo chmod 666 /var/www/html/exam_system/exam_system.db

# 4. 重启Apache
sudo systemctl restart apache2

# 5. 访问系统
浏览器打开：http://localhost/exam_system
```

### 方案三：PHP内置服务器（开发测试）
```bash
cd exam_system
php -S 0.0.0.0:8000
```

## 📊 性能指标

### SQLite性能
- **适用规模**：<1000并发用户
- **数据容量**：<10GB数据
- **并发读取**：优秀
- **并发写入**：一般（单线程写入）

### 性能优化建议
1. **添加索引**
```sql
CREATE INDEX idx_user_username ON users(username);
CREATE INDEX idx_exam_questions ON exam_questions(exam_id);
CREATE INDEX idx_submissions ON submissions(user_id, exam_id);
```

2. **启用WAL模式**（提升并发性能）
```sql
PRAGMA journal_mode=WAL;
```

3. **迁移到MySQL**（大规模应用）
- 支持更高并发
- 更好的写入性能
- 支持主从复制

## 🐛 常见问题

### Q1: 数据库文件在哪里？
A: Docker部署在 `/var/www/html/data/exam_system.db`，传统部署在项目根目录。

### Q2: 如何重置系统？
A: 删除数据库文件，系统会自动重新创建并初始化。

### Q3: 忘记管理员密码怎么办？
A: 删除数据库文件重新初始化，或使用SQLite工具修改密码。

### Q4: 如何添加更多管理员？
A: 注册普通账号后，使用SQLite工具修改role字段为'admin'。

### Q5: 支持多少并发用户？
A: SQLite适合<50并发用户，更大规模建议迁移到MySQL。

### Q6: 如何备份数据？
A:
```bash
# Docker环境
docker-compose exec exam-system sqlite3 /var/www/html/data/exam_system.db .dump > backup.sql

# 传统环境
sqlite3 exam_system.db .dump > backup.sql
```

### Q7: 如何恢复数据？
A:
```bash
# Docker环境
cat backup.sql | docker-compose exec -T exam-system sqlite3 /var/www/html/data/exam_system.db

# 传统环境
sqlite3 exam_system.db < backup.sql
```
